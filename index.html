01

CSS Animacije

CSS animacije omogućavaju glatke vizuelne prijelaze bez potrebe za JavaScriptom. Koriste se @keyframes pravila, transition svojstva i transform funkcije za kreiranje efekata koji se izvršavaju na GPU-u, čime se postiže izuzetno glatko iskustvo pri 60 FPS.

Hover Efekti

Hover efekti se aktiviraju kada korisnik pređe mišem preko elementa. Koriste CSS :hover pseudo-klasu u kombinaciji sa transition svojstvom za glatke prijelaze. Ovi efekti pružaju vizuelni feedback korisniku i poboljšavaju interaktivnost stranice.

↑

Lift Efekat

Transform: translateY(-10px) uz box-shadow za efekat lebdenja. Koristi transition za glatki prijelaz.

transform: translateY(-10px);
box-shadow: 0 20px 40px rgba(0,0,0,0.3);
⬡

Scale Efekat

Transform: scale(1.05) povećava element za 5%. Vizuelno naglašava element pod kursorom.

transform: scale(1.05);
transition: transform 0.3s ease;
✦

Glow Efekat

Box-shadow sa neon bojom kreira efekat sijanja. Kombinuje se sa border-color tranzicijom.

box-shadow: 0 0 30px rgba(139,92,246,0.5);
border-color: #8b5cf6;
↻

Rotate Efekat

Transform: rotate(5deg) rotira element za 5 stepeni. Suptilna rotacija privlači pažnju.

transform: rotate(5deg);
transition: transform 0.3s cubic-bezier(.4,0,.2,1);
◻

Border Animacija

Gradient border koji se pojavljuje na hover koristeći background-clip i gradient pozadine.

background: linear-gradient(...);
background-clip: padding-box;
◎

Blur Efekat

Backdrop-filter: blur() za glassmorphism efekat. Popularan u modernom UI dizajnu.

backdrop-filter: blur(20px);
background: rgba(255,255,255,0.1);

Button Animacije

Animirani dugmadi pružaju jasan vizuelni feedback korisniku. Koriste kombinaciju transform, box-shadow, background tranzicija i ::before/::after pseudo-elemenata za kreiranje naprednih efekata.

Ripple: JavaScript detektuje klik koordinate i kreira kružnu animaciju koristeći CSS radial-gradient. Shine: Pseudo-element sa linear-gradient se pomjera preko dugmeta koristeći @keyframes. Fill: Pozadina se animira od dna prema vrhu koristeći ::before element sa scaleY tranzicijom. Gradient Shift: Background-position se animira na hover za efekat pomicanja gradijenta.

Loading Animacije

Loading indikatori koriste @keyframes animacije za beskonačno ponavljanje (animation-iteration-count: infinite). Koriste se za vizuelni feedback dok korisnik čeka učitavanje sadržaja. Ove animacije trebaju biti lagane i ne smiju opterećivati CPU/GPU.

Ring Spinner border + border-top-color + @keyframes spin { transform: rotate(360deg) }
Bouncing Dots @keyframes bounce sa animation-delay za svaki dot element
Pulse @keyframes pulse { scale(0) → scale(1) + opacity: 0 }
Wave Bars scaleY animacija sa staggered animation-delay
Orbit @keyframes orbit — rotacija tačke oko centra 360°
Gradient Spin conic-gradient + @keyframes rotate + border-radius: 50%

CSS Tranzicije

Tranzicije definišu kako se promjene CSS svojstava odvijaju tokom vremena. Ključna svojstva su: transition-property (šta se animira), transition-duration (koliko traje), transition-timing-function (brzinska kriva) i transition-delay (odgoda).

02

JavaScript / GSAP

GSAP (GreenSock Animation Platform) je najmoćnija JavaScript biblioteka za web animacije. Pruža preciznu kontrolu nad timing-om, easing funkcijama i sekvenciranjem animacija. GSAP koristi requestAnimationFrame i optimizira performanse automatski. Podržava animaciju bilo kojeg numeričkog CSS svojstva, SVG atributa, pa čak i custom objekata.

Pomjeranje Elemenata

GSAP metode gsap.to(), gsap.from() i gsap.fromTo() omogućavaju precizno pomjeranje elemenata. Svojstva poput x, y, rotation, scale se animiraju kroz transformer, što je GPU-ubrzano.

gsap.to()
gsap.from()
gsap.fromTo()

gsap.to(): Animira element OD trenutnog stanja DO zadatih vrijednosti. gsap.from(): Animira element OD zadatih vrijednosti DO trenutnog stanja. gsap.fromTo(): Definiše i početno i krajnje stanje animacije.

Timeline Animacije

gsap.timeline() omogućava sekvenciranje više animacija u jednu kontrolabilnu cjelinu. Svaka animacija u timeline-u se automatski nadovezuje na prethodnu, ali se može i preklapati koristeći position parametre poput "<0.5" ili "-=0.3". 

const tl = gsap.timeline({ paused: true });

tl.from(".circle", { scale: 0, duration: 0.5 })
  .from(".square", { x: -100, opacity: 0, duration: 0.5 }, "-=0.2")
  .from(".triangle", { y: 50, rotation: -180, duration: 0.6 }, "-=0.3")
  .from(".diamond", { scale: 0, rotation: 360, duration: 0.5 })
  .to(".circle, .square", { y: -30, stagger: 0.1, duration: 0.4 });

Stagger Animacije

Stagger parametar u GSAP-u omogućava kaskadno pokretanje animacije za grupu elemenata sa zadanim vremenskim razmakom. Ovo stvara efekat "talasa" koji je vizuelno vrlo privlačan. Podržava i napredne konfiguracije poput stagger: { each: 0.1, from: "center" }.

Kompleksni Efekti

GSAP omogućava kreiranje kompleksnih animacija kombinovanjem više tehnika: morphing oblika, animacija teksta po slovima, 3D transformacije i dinamičke interakcije sa korisnikom.

SVG Animacije

SVG (Scalable Vector Graphics) animacije omogućavaju animiranje vektorske grafike direktno u browseru. SVG elementi se mogu animirati putem CSS-a, SMIL-a ili JavaScripta. Posebno su korisni za line-drawing efekte (stroke-dasharray / stroke-dashoffset), animirane ikone i ilustracije koje se prilagođavaju svim rezolucijama.

Line Drawing Efekat

Tehnika "line drawing" koristi SVG stroke-dasharray i stroke-dashoffset svojstva. Postavljanjem dasharray na ukupnu dužinu putanje i animiranjem dashoffset-a od te dužine do nule, stvara se efekat crtanja linije.

Scroll Efekti

Scroll-bazirane animacije se aktiviraju na osnovu pozicije skrolovanja stranice. GSAP ScrollTrigger plugin omogućava preciznu kontrolu — definisanje tačnog trenutka kada animacija počinje i završava, "pinning" elemenata, parallax efekte i "scrub" animacije koje su direktno vezane za poziciju skrola.

Reveal Animacije na Skrolovanje

Elementi se pojavljuju sa različitim animacijama dok korisnik skroluje stranicu. ScrollTrigger definiše "trigger" element i "start" poziciju (npr. "top 80%" znači da animacija počinje kada vrh elementa dostigne 80% visine viewporta).

🎨

Fade Up

Element se pojavljuje pomjerajući se prema gore sa opacity tranzicijom. Najčešći reveal efekat.

gsap.from(el, { y: 60, opacity: 0, scrollTrigger: { trigger: el, start: "top 80%" } })
🚀

Slide from Left

Element klizi s lijeve strane u svoj položaj. Koristi x translaciju sa ScrollTrigger-om.

gsap.from(el, { x: -100, opacity: 0 })
⚡

Slide from Right

Element klizi s desne strane. Simetričan efekat koji se koristi za alternating layouts.

gsap.from(el, { x: 100, opacity: 0 })
💎

Scale In

Element se pojavljuje rastom od nule do pune veličine. Dramatičan efekat za ključne elemente.

gsap.from(el, { scale: 0, opacity: 0 })
🔄

Rotate In

Element se rotira dok se pojavljuje. Kombinuje rotaciju i opacity za dinamičan ulaz.

gsap.from(el, { rotation: -15, opacity: 0 })
✨

Flip In

3D flip efekat koristeći rotateX. Stvara utisak da se element okreće kao kartica.

gsap.from(el, { rotateX: -90, opacity: 0 })

Parallax Efekat

Parallax efekat stvara iluziju dubine tako što se različiti slojevi stranice pomjeraju različitim brzinama pri skrolovanju. Prednji slojevi se pomjeraju brže od pozadinskih, što stvara trodimenzionalni utisak. Implementira se putem ScrollTrigger scrub parametra.

06

Performanse Demo

Performanse animacija su ključne za glatko korisničko iskustvo. Animacije koje koriste svojstva poput transform i opacity se izvršavaju na GPU-u (compositor thread) i ne uzrokuju layout ili paint operacije. Nasuprot tome, animiranje width, height, top, left i sličnih svojstava uzrokuje skupe layout preračunavanja (reflow).

Loša vs Dobra Animacija

Uporedni prikaz dviju animacija sa identičnim vizuelnim rezultatom, ali drastično različitim performansama. Lijeva strana koristi loša svojstva (left, top, width, height), dok desna koristi dobra svojstva (transform, opacity).

Loše

Koristi: left, top, width, height

  • Pokreće Layout preračunavanje
  • Pokreće Paint ponovno iscrtavanje
  • Zauzima Main Thread
  • Uzrokuje jank (trzanje)
  • Nije GPU-ubrzano
JS → Style → Layout → Paint → Composite

  • Preskače Paint fazu

Animacija Boja: background-color vs Overlay

Animiranje background-color pokreće Paint fazu. Bolji pristup je korištenje preklapajućeg elementa sa opacity tranzicijom i fiksnom bojom — browser samo mijenja prozirnost na GPU-u bez ponovnog iscrtavanja piksela.

Optimizacijski Savjeti

Ključni principi za performantne web animacije koji osiguravaju glatko korisničko iskustvo.

01

Koristi transform & opacity

Ova dva CSS svojstva su jedina koja se mogu animirati bez pokretanja Layout i Paint faza rendering pipeline-a. Browser ih obrađuje na GPU-u (compositor thread), što omogućava 60 FPS animacije.

/* Dobro ✅ */
transform: translateX(100px);

/* Loše ❌ */
left: 100px;
02

will-change Svojstvo

will-change informiše browser unaprijed koja svojstva će se mijenjati, omogućavajući mu da izvrši optimizacije. Ali se ne smije pretjerivati — previše will-change deklaracija može zapravo pogoršati performanse.

/* Koristi samo kad je potrebno */
will-change: transform, opacity;

/* Ukloni nakon animacije */
will-change: auto;
03

requestAnimationFrame

Za JavaScript animacije uvijek koristi requestAnimationFrame umjesto setTimeout ili setInterval. rAF sinhronizuje animacije sa refresh rate-om ekrana i pauzira kad tab nije aktivan.

function animate() {
  // animacija logika
  requestAnimationFrame(animate);
}
04

Izbjegavaj Layout Thrashing

Čitanje i pisanje DOM svojstava naizmjenično uzrokuje "layout thrashing" — browser mora konstantno preračunavati layout. Grupiši čitanja i pisanja zasebno.

/* Loše - layout thrashing ❌ */
el.style.width = el.offsetWidth + 10 + 'px';

/* Dobro - batchovano ✅ */
const w = el.offsetWidth;
el.style.width = w + 10 + 'px';
05

Reduce, Reuse, Recycle

Minimiziraj broj animiranih elemenata. Koristi CSS containment (contain: layout) da ograničiš područje koje browser mora preračunati. Recikliraj animacije umjesto da kreiraš nove.

/* CSS Containment */
.animated-section {
  contain: layout style paint;
}
06

prefers-reduced-motion

Poštuj korisnikove postavke pristupačnosti. Medijski upit prefers-reduced-motion detektuje da li je korisnik zatražio smanjeno kretanje u sistemskim postavkama.

@media (prefers-reduced-motion: reduce) {
  * { animation-duration: 0.01ms !important; }
}

FPS Monitor

Demonstracija razlike u FPS-u (frames per second) između optimizirane i neoptimizovane animacije. Zelena zona (50-60 FPS) označava glatku animaciju, žuta (30-50 FPS) je prihvatljiva, a crvena (ispod 30 FPS) ukazuje na probleme sa performansama.
